`PortScanner` probes `127.0.0.1` (and `::1` as a fallback) for listening TCP services using non-blocking `Socket.connect()` calls. No special permissions are required — any app can attempt a TCP connection to localhost. Open ports are a direct indicator of a running proxy process; subsequent modules (`Socks5Probe`, `ClashAPIProbe`, `XrayAPIDetector`) then interrogate each open port to determine what service is listening.

## How it works

Each port probe opens a new `Socket`, calls `socket.connect(InetSocketAddress(host, port), timeoutMs)`, and records success or timeout. The response time in milliseconds is stored alongside the port number. IPv4 is tried first; if it fails, the same port is tried on IPv6 `::1`.

```kotlin
data class OpenPort(
    val port: Int,
    val responseMs: Long
)
```

## Quick scan vs full scan

<AccordionGroup>
  <Accordion title="Quick scan — scanKnownPorts()">
    Iterates the `KNOWN_PORTS` list sequentially with a 300 ms timeout per port. Completes in under one second. Use this as the default scan mode.

    ```kotlin
    suspend fun scanKnownPorts(
        timeoutMs: Int = 300,
        onProgress: (Float) -> Unit = {}
    ): List<OpenPort>
    ```
  </Accordion>
  <Accordion title="Full scan — scanFullRange()">
    Scans all 65,535 ports using batched coroutines. The default `parallelism` of 200 means 200 ports are probed concurrently per batch, using a 50 ms timeout. On a typical Android device this completes in 3–5 seconds.

    ```kotlin
    suspend fun scanFullRange(
        timeoutMs: Int = 50,
        parallelism: Int = 200,
        onProgress: (Float) -> Unit = {}
    ): List<OpenPort>
    ```

    Results are sorted by port number before returning.
  </Accordion>
</AccordionGroup>

<Tip>
Use the quick scan first. Switch to the full range scan only when the quick scan finds nothing but indirect signs (MTU anomaly, VPN transport flag) still suggest a proxy is running on a non-standard port.
</Tip>

## IPv6 localhost scanning

Every port is also probed on `::1` when the IPv4 attempt fails. Some VPN clients bind exclusively to the IPv6 loopback address, particularly sing-box configurations with `"listen": "::"`. The `probePort(HOST_V6, port, timeoutMs)` fallback ensures these are not missed.

## When to use each mode

| Situation | Recommended mode |
|-----------|-----------------|
| Routine scan, VPN client active | Quick scan (default) |
| Quick scan finds nothing, but `TRANSPORT_VPN` is set | Full range scan |
| Investigating an unknown app | Full range scan |
| Background periodic check | Quick scan |
